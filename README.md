# 한국 자격증 기출문제 풀이 플랫폼 (MVP)

코딩 경험이 없어도 실행할 수 있도록 만든 모바일 퍼스트 웹앱입니다.

## 현재 구현된 기능

- 문제 풀이: 자격증/연도/회차 선택, 순차/랜덤 풀이
- 채점: 보기 클릭 즉시 정오답 + 해설 표시
- 오답 정리: 틀린 문제 자동 분류
- 오답노트: 문제별 메모 작성/저장 (브라우저 저장)

## 파일 구성

- `index.html`: 화면 구조
- `styles.css`: 모바일 퍼스트 UI 스타일
- `sample-data.js`: 샘플 기출 데이터
- `app.js`: 기능 로직

## 실행 방법

### A) 문제 풀이만 사용 (샘플 데이터)

1. `index.html` 더블클릭으로 실행 가능

### B) Q-Net API 연동 사용 (권장)

1. 터미널에서 프로젝트 폴더로 이동
2. `node server.js` 실행
3. 브라우저에서 `http://localhost:3000` 접속
4. ServiceKey 입력 후 `공개문제 목록 불러오기` 클릭

> `Failed to fetch`가 나오면 대부분 `index.html`을 파일로 직접 연 경우입니다.  
> 반드시 `node server.js` 후 `http://localhost:3000`으로 접속하세요.

## 사용 방법

1. 자격증/연도/회차 선택
2. `문제 풀이 시작` 클릭
3. 보기 버튼 클릭 시 바로 채점
4. 아래 메모 칸에 오답노트 작성
5. `오답만 보기`로 틀린 문제 재풀이

## 다음 단계 (Q-Net 연동)

현재는 샘플 데이터로 동작하며, 1단계 연동(공개문제 목록 조회)은 앱에 추가되어 있습니다.

### 1단계: ServiceKey 발급 후 목록 조회

1. 공공데이터포털에서 **「국가자격 공개문제 조회 서비스」** 활용신청 후 `ServiceKey` 발급  
   (data.go.kr 검색, API ID: B490007 / `getOpenQstList`)
2. 앱 상단의 `Q-Net 공개문제 API 연동 (1단계)` 카드에서 키 입력
3. `공개문제 목록 불러오기` 클릭
4. 성공 시 게시물 목록(제목/게시물ID/종목)이 표시됨

### API 실패 시 체크

- `Failed to fetch`: `node server.js` 실행 후 `http://localhost:3000` 접속 여부 확인
- `HTTP 401/403`: ServiceKey 오류 또는 **공개문제 API 미신청**
- `Q-Net HTTP 404`: 예전 잘못된 주소 사용 시 발생 (현재 버전은 수정됨)
- `공공데이터 API HTTP 5xx`: 서버 일시 장애, 잠시 후 재시도

### ServiceKey 보안

- `.env.example`를 복사해 `.env` 파일을 만들고 키를 넣을 수 있습니다.
- 키가 채팅/스크린샷에 노출되었다면 공공데이터포털에서 **재발급**하는 것을 권장합니다.

## 2단계 이후 계획

실제 큐넷 공개문제 연동을 위해:

1. 공공데이터포털에서 Q-Net 오픈 API 키 발급
2. 공개문제 상세 API 호출
3. 문제 데이터 정규화 후 앱에 주입
4. API 키 보호를 위해 간단한 백엔드 추가

원하면 다음 작업으로 Q-Net API 연결 버전을 이어서 구현합니다.
